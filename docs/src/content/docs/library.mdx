S-Bank is the DSP, not the modules. Everything lives in one header,
`modules/rack/src/dsp/SBankDSP.hpp`, under namespace `sbank`. There is **no VCV-Rack-SDK
dependency**: the header pulls in `<algorithm>`, `<array>`, `<cmath>`, `<cstdint>`, and
`<cstddef>` and nothing else. It compiles and runs headless, so you can embed it in a Rack
module, a plugin, a command-line renderer, or a test harness.

```cpp
#include <sbank/sbank_dsp.hpp>   // canonical, stable include
```

That is the whole installation: copy the header into your build, add its directory to your
include path (`-I`), and use the types below. `<sbank/sbank_dsp.hpp>` is a thin forwarding
header over the real implementation in `modules/rack/src/dsp/SBankDSP.hpp` (inside this repo
the demos include the latter directly). The demo modules `Strike` and `VactrolLPG` are thin
Rack adapters over exactly these classes — they exist to prove the library, not to be it.

The header carries its own version: `SBANK_DSP_VERSION_MAJOR` / `_MINOR` / `_PATCH` (and the
combined `SBANK_DSP_VERSION`) let a vendored copy branch on the library version.

<Aside type="tip" title="Configure with a typed struct">
  Every voice takes a named parameter struct — `setParams(const LpgParams&)`,
  `setParams(const StrikeParams&)` — so each field is named and unit-annotated, and a swapped
  argument is a compile error rather than a silent wrong sound. Enum overloads
  (`setParams(LpgMode, ...)`) and the original positional `setParams(...)` of bare floats are
  kept as shims for existing call sites, but new code should reach for the struct.
  `getParams()` reads the active values back.
</Aside>

## VactrolLpg — the Buchla-292 vactrol gate

`sbank::VactrolLpg` is the full vactrol low-pass gate: a control path (CV smoothing plus the
Parker &amp; D'Angelo opto-current model), the vactrol resistance model with its slow attack and
bloomed release, and a state-space audio cell with optional oversampling.

<Steps>

1. **Construct** with the sample rate (defaults to `48000.f`):
   `sbank::VactrolLpg lpg(sampleRate);`

2. **Configure** with `setParams(const LpgParams&)` — a named struct (or the
   `setParams(LpgMode, ...)` overload).

3. **Run** one sample at a time through `processSample(audioIn, cvIn)`.

</Steps>

`LpgParams` is the configuration struct; the table is its fields, and the defaults shown are
exactly what a default-constructed `LpgParams` holds:

| Field | Type | Meaning | Default · range |
|-------|------|---------|-----------------|
| `mode` | `LpgMode` | `LpgMode::Both` (LPG), `Vca`, or `Lowpass` | `Both` |
| `resonance` | `float` | Audio-cell resonance | `0.2` · 0.0 – 1.0 (clamped) |
| `cvOffset` | `float` | DC added to the incoming CV before the control model | `0.0` volts |
| `drive` | `float` | `tanh` drive into the cell; `0` disables the nonlinearity | `1.0` · ~0.0 – 8.0 |
| `oversample` | `uint32_t` | Requested oversampling; `oversampleFactor()` snaps it to 1× / 2× / 4× | `2` · 1, 2, or 4 |

```cpp
sbank::VactrolLpg lpg(48000.f);

sbank::LpgParams p;               // starts at the defaults above
p.mode = sbank::LpgMode::Both;
p.resonance = 0.2f;
p.oversample = 2;                 // 2x
lpg.setParams(p);
// equivalently: lpg.setParams(sbank::LpgMode::Both, 0.2f, /*cvOffset*/ 0.f, /*drive*/ 1.f, 2);

for (int i = 0; i < 48000; ++i) {
    float x  = std::sin(2.f * sbank::kPi * 220.f * i / 48000.f); // audio, nominal +-1.0
    float cv = i < 12000 ? 8.f : 0.f;                            // CV in volts: open, then close
    float y  = lpg.processSample(x, cv);                         // one sample out
    // y is nominal +-1.0
}
```

`lpg.lastResistance()` exposes the current vactrol resistance (ohms) if you want to drive a
meter or a scope. Call `lpg.reset()` to clear the cell state, the smoother, and the vactrol
back to its dark (fully closed) resistance.

To change the *analog character* — the capacitor values, the vactrol power law and its time
constants — pass an `LpgComponents` to `lpg.setComponents(...)` (`getComponents()` reads it
back). The defaults are the datasheet/reference values the golden regression pins, so the
out-of-the-box sound is unchanged until you deliberately override them.

## StrikeCore — the clean, zero-bleed gate

`sbank::StrikeCore` is the inverse philosophy: a shaped envelope drives a clean state-variable
gate that bleeds nothing when closed, with an always-available analog-drift layer.

<Steps>

1. **Construct** with the sample rate and an optional 64-bit seed for the drift RNGs:
   `sbank::StrikeCore strike(sampleRate);`

2. **Configure** the voice with `setParams(const StrikeParams&)`, and optionally the drift with
   `setImperfection(const StrikeImperfectionConfig&)`.

3. **Run** one sample through `processSample(audioIn, ctrl01, decayMod, hitV)`.

</Steps>

`StrikeParams` is the configuration struct (default-constructed values in the last column):

| Field | Type | Meaning | Default · range |
|-------|------|---------|-----------------|
| `open` | `float` | Gate floor — minimum openness even with no hit | `0.0` · 0.0 – 1.0 |
| `decay` | `float` | Envelope decay; longer ring as it rises | `0.4` · 0.0 – 1.0 |
| `material` | `float` | Hard → soft: sets attack time, cutoff ceiling, and level | `0.0` · 0.0 – 1.0 |

`processSample(float audioIn, float ctrl01, float decayMod, float hitV)`:

| Arg | Type | Meaning | Range / units |
|-----|------|---------|---------------|
| `audioIn` | `float` | Audio to gate | nominal +-1.0 |
| `ctrl01` | `float` | Direct openness control, summed into the envelope target | 0.0 – 1.0 |
| `decayMod` | `float` | Additive decay modulation; `0` for none | ~0.0 – 1.0 units |
| `hitV` | `float` | Trigger; a rising edge crossing `0.25` fires the envelope | volts (e.g. `0` / `5`+) |

```cpp
sbank::StrikeCore strike(48000.f);

sbank::StrikeParams p;                      // open=0, decay=0.4, material=0 by default
p.decay = 0.5f;
strike.setParams(p);                        // or: strike.setParams(0.f, 0.5f, 0.f);

for (int i = 0; i < 96000; ++i) {
    float hit = i < 10 ? 5.f : 0.f;        // a brief trigger edge, in volts
    float y   = strike.processSample(1.f, 0.f, 0.f, hit); // IN held at DC -> percussive ping
    // y is nominal +-1.0
}
```

Feeding a DC level into `audioIn` (here `1.f`) with no other source makes a `hitV` edge emit
the raw envelope — a percussive ping — which is exactly how Strike's `IN` jack is normalled in
the Rack adapter. To turn on the seeded drift and pink-noise floor, set the fields of a
`StrikeImperfectionConfig` and pass it:

```cpp
sbank::StrikeImperfectionConfig dirt;   // enabled=false, noiseAmp=1e-4, driftAmount=1.0
dirt.enabled = true;
strike.setImperfection(dirt);           // or the shim: strike.setImperfection(true, 1e-4f, 1.f);
```

`reset()` clears the envelope, gate, pitch tracker, and drift state.

## Signal conventions

These are the conventions the Rack adapters rely on, and they are the contract for any host:

- **Audio is nominal `+-1.0`.** The Rack wrappers convert at the jack — they divide the
  `+-5 V` Eurorack signal by 5 on the way in and multiply by 5 on the way out. Inside the
  library, audio is unit-scaled.
- **CV is in volts.** `VactrolLpg::processSample` takes `cvIn` directly in volts (the demo
  feeds `8 V` to open the gate); the control model clamps it internally before smoothing.
- **Strike's control inputs are pre-normalized.** `ctrl01` and the `decayMod` are `0..1`-scale
  values, not raw volts — the adapter does the `/10` mapping and attenuverter math before the
  call. Only `hitV` is a voltage: a rising edge through the `0.25` threshold fires the strike.

## Real-time-safety contract

The per-sample path is written to be audio-thread safe. Honoring this contract is on you:

- **`processSample` does not allocate, lock, or throw.** No heap, no mutexes, no exceptions on
  the audio path. State lives in fixed-size members and `std::array`s sized at compile time.
- **Call `setSampleRate(...)` off the audio thread.** It recomputes smoothing and filter
  coefficients; treat it as setup or a control-thread event (the adapters call it from
  `onSampleRateChange`), not per block.
- **`setParams(...)` is cheap and audio-thread safe** — the adapters call it every sample. It
  only stores values and (for the vactrol) maps the oversample factor.
- **`reset()` clears all internal state.** Use it on patch reset or voice steal to silence
  ringing tails and return the vactrol to its dark resistance. It does not allocate.
- **Non-finite inputs are scrubbed.** Every `processSample` replaces a NaN/Inf input with `0`
  before it can reach the recursive state, so a bad sample upstream can't poison the filter.

<Aside type="note">
  Each voice also offers `processBlock(...)`, a convenience that loops `processSample` over
  length-`n` buffers — a trivially-correct wrapper, not a vectorised fast path. The per-sample
  call is the exact one (it is what keeps the state-space and oversampled cells exact); reach
  for `processBlock` only where a buffer-at-a-time call site reads more cleanly.
</Aside>

## The building blocks

`VactrolLpg` and `StrikeCore` are themselves compositions. The same header exposes the smaller
pieces, so you can wire them into a new voice instead of reaching for the prebuilt gates:

| Block | Role |
|-------|------|
| `ControlPath` / `ControlCoeffs` | CV smoothing &amp; the opto-current model — volts in, vactrol drive current out |
| `VactrolModel` | The vactrol cell itself: drive current in, time-varying resistance out (slow attack, bloomed release) |
| `AudioPath` / `Cell292` | The state-space Buchla-292 audio cell, with resonance and `tanh` drive |
| `Oversampler` / `HalfbandStage` | 2×/4× polyphase half-band oversampling around any per-sample nonlinearity |
| `Envelope` | Strike's dual-rate (fast + slow) envelope with pitch-aware decay |
| `Gate` | The clean, zero-bleed state-variable low-pass gate |
| `Material` | Maps a single `0..1` "material" into attack time, cutoff ceiling, and level |
| `PitchTracker` | A lightweight zero-crossing pitch estimator (used to scale decay) |
| `PinkNoise`, `XorShift`, `SplitMix64` | The RNG / noise primitives behind the seeded analog drift |

Most blocks share the same shape — `setSampleRate`, `reset`, and a `process(...)` — so they
compose cleanly. To build a new voice, pick the cell you want (the vactrol audio path, or the
clean gate), drive it from a control source (the opto model, an envelope, or your own CV), and
wrap the nonlinear inner step in the `Oversampler` if you need the alias rejection. Seed an
`XorShift` or `SplitMix64` to give two instances reproducibly different analog character.

<Aside type="tip">
  The same pieces, wired two ways, give the two demo voices: <strong>VactrolLpg</strong> =
  ControlPath → VactrolModel → AudioPath; <strong>StrikeCore</strong> = Envelope (+ PitchTracker
  + Material) → Gate, with the drift RNGs layered on top.
</Aside>
