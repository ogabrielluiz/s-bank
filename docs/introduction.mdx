**S-Bank** is the signal bank — a small set of virtual-analog instruments for [VCV Rack 2](https://vcvrack.com),
built on a header-only C++ DSP library and wrapped in the **Sam-e / S-** visual system.

The library is the product; the modules are the proof. Two ship today:

<CardGroup cols={2}>
  <Card title="Strike" icon="bolt" href="/modules/strike">
    A dual, clean, **zero-bleed** envelope-driven low-pass gate. Sharp percussive
    pings, a continuous material morph, and an always-on analog drift.
  </Card>
  <Card title="Vactrol LPG" icon="wave-square" href="/modules/vactrol-lpg">
    A Buchla-292-style **vactrol** low-pass gate — dirty and resonant, with the lag
    and bloom of a real opto-isolator. Low-pass / VCA / combined.
  </Card>
</CardGroup>

## What makes them analog

Both modules model the *circuit*, not just a filter sweep:

- **Vactrol LPG** is the Parker & D'Angelo state-space Buchla-292 cell — the opto-isolator's
  slow attack and long, bloomed release are inherent to the model, not an envelope bolted on.
- **Strike** is the inverse: a deliberately clean, gate that does **not** leak when closed,
  driven by a shaped envelope — with a whisper of seeded analog drift so "clean" never tips
  into "sterile."

<Note>
  The DSP is native, header-only C++ (`modules/rack/src/dsp/SBankDSP.hpp`) with no Rack-SDK
  dependency, so it tests and runs headless. See [Develop](/development).
</Note>
