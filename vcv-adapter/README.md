# VCV adapter (Phase 7)

A thin C++ `rack::Module` that links the Rust `vactrol-core` staticlib over the
C ABI in `crates/vactrol-core/src/ffi.rs`. This is the **last, optional** phase:
nothing here blocks the core pipeline, which builds and tests with no Rack SDK.

> Status: scaffold. The C ABI header is generated and accurate, and the module
> source is written against the Rack v2 API, but it is **not built or verified
> here** because the Rack SDK and a Rack-targeting C++ toolchain are not present
> in this environment. Treat it as a faithful skeleton, not a tested artifact.

## Layout

- `vactrol_core.h` -- C ABI header, generated by `cbindgen` from the Rust core
  (regenerate: `cd ../crates/vactrol-core && cbindgen --config cbindgen.toml --output ../../vcv-adapter/vactrol_core.h`).
- `src/plugin.{hpp,cpp}` -- plugin entry points.
- `src/VactrolLPG.cpp` -- the module: owns one opaque core instance per polyphony
  channel and forwards `process()` per sample.
- `plugin.json` -- Rack plugin manifest.
- `Makefile` -- links the prebuilt `libvactrol_core.a`.
- `res/` -- panel SVG (to be added).

## Build

```sh
# 1. Build the Rust core staticlib.
(cd ../crates/vactrol-core && cargo build --release)

# 2. Build the plugin against the Rack SDK.
make RACK_DIR=/path/to/Rack-SDK
```

## FFI contract

The boundary is a flat C ABI: `create` / `destroy` / `set_sample_rate` /
`set_params` / `process_sample` / `process_block`. Every entry point is
panic-safe (`catch_unwind`) and allocation-free on the audio thread; the only
heap activity is the create/destroy pair.

## Notes / known friction

- The Rust core is currently scalar (one voice per call). When it gains a
  lane-parametric SIMD (`float_4`) path, map four channels onto one core call in
  `process()` instead of the per-channel loop.
- Community reports getting Rust-in-Rack working on macOS/ARM but hitting
  linking friction on Windows MSYS2/MinGW. Verify on your target platforms.
- Licensing: the clean-room core is `MIT OR Apache-2.0`. Shipping inside Cardinal
  or as a non-exception VCV plugin would make the whole plugin GPLv3; choose
  deliberately before release.
